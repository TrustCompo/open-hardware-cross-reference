# Contributing

Thanks for helping improve TrustCompo Open Hardware Cross Reference.

This project is intended for hardware engineers, component engineers, and
sourcing teams who need a practical first-pass view of lifecycle-affected parts
and possible replacement candidates.

## Useful Contributions

- correct an inaccurate cross-reference or fit score
- add a missing replacement candidate with evidence
- clarify parameter, package, pinout, or lifecycle differences
- improve validation notes for a part page
- report broken links or stale source references

## Evidence Expectations

When proposing a correction, please include enough public evidence for another
engineer to review the claim:

- original manufacturer part number
- candidate manufacturer part number
- manufacturer and package
- relevant datasheet or manufacturer page
- affected parameter or lifecycle field
- reason the current page is wrong, incomplete, or ambiguous

Avoid submitting blanket equivalence claims without validation context. A
candidate can be useful in a shortlist even when it is not a drop-in replacement,
as long as the limits are clearly stated.

## Review Boundary

Published pages are not approval documents. Changes should help engineers start
review work faster while preserving clear warnings about remaining validation
steps.
