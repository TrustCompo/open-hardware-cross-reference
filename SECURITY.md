# Security Policy

## Scope

This repository publishes public cross-reference pages, static site assets, and
machine-readable exports for component review workflows. Security-sensitive
reports may include:

- materially incorrect replacement guidance that could create electrical,
  thermal, compliance, or sourcing risk
- supply-chain or lifecycle data that appears to be misleading or stale
- links that point to unexpected or unsafe destinations
- vulnerabilities in the static publishing workflow

## Reporting a Vulnerability

Please report sensitive issues through GitHub private vulnerability reporting
if it is available for this repository. If private reporting is not available,
open a GitHub issue only when the details can be discussed publicly without
creating risk for downstream users.

For ordinary data corrections, missing alternatives, or unclear validation
boundaries, please use the issue templates in this repository.

## Validation Boundary

The pages in this repository are shortlist and review aids. They are not design
approval documents, engineering sign-off, manufacturer guarantees, or legal
warranty statements. Every replacement candidate still requires datasheet,
package, pinout, parametric, lifecycle, compliance, and application-level
validation before use in production.
